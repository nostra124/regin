# FEAT-023 — Session archival + transcript capture

## Description
**As** regin
**I want** my own past working sessions retained as readable transcripts with an LLM
summary
**So that** the identity holds *what was done and decided* — the core of the
self-improving memory — and can recall across sessions.

## Implementation
- **Session lifecycle:** when a working session starts (chat / task / operator run),
  open a `sessions` row (`kind`, `host`, `title`, `started_at`, `state='open'`). On
  close: write the readable `transcripts` row (markdown), set `ended_at`,
  `message_count`, `token_count`, `state='closed'`.
- **LLM session summary:** on close, generate `sessions.summary` (a compact summary
  for cross-session recall); bounded in tokens; fails safe (on error, store the
  transcript without a summary and log).
- **Episode link:** emit a `kind='session'` (or per-source `chat|task|...`) `episode`
  (`state='new'`, `ref_id` = session id, `host`) so the consolidation pass (FEAT-024)
  picks the session up as raw material.
- Capture is the **cheap write** path — no consolidation here; transcripts go in as
  `TEXT`, indexed by `transcripts_fts` via the FEAT-021 triggers.
- `host` is set to the machine the session ran on (NULL only for host-agnostic work).

## Acceptance Criteria
1. A started→closed session produces one `sessions` row and one `transcripts` row,
   with `message_count`/`token_count` populated and `state` transitioning
   `open→closed`.
2. `sessions.summary` is generated by the LLM on close and is bounded; an LLM failure
   still persists the transcript (fail-safe) and logs.
3. A closed session emits a `new` episode referencing it, ready for consolidation.
4. The transcript is searchable via `transcripts_fts`.
5. Unit-tested with a fake LLM and a fake clock.
